# Data

This directory holds two kinds of data:

- **`tiny/`** — a tiny, ~1 MB subset (~1500/250/250 train/val/test rows)
  that is **committed** to the repo and used by the smoke test
  (`make smoke`). It was produced by `scripts/make_tiny_subset.py` from the
  full training set with a fixed seed; the script over-samples
  identity-mentioning rows so that per-identity bias AUCs are computable on
  such a small sample.
- **`full/`** — the actual Jigsaw competition CSVs, which are **not**
  committed (gitignored). You must download them separately.
- **`processed/`** — pre-tokenised memmap files generated by
  `python -m toxic_classifier.data.prepare`. Also gitignored.
- **`splits/`** — saved train/val split indices (numpy `.npy`). Gitignored.

## Fetching the full Jigsaw data

The competition data is available at:
https://www.kaggle.com/c/jigsaw-unintended-bias-in-toxicity-classification/data

The simplest path is the Kaggle CLI (you must accept the competition rules
on Kaggle first, then create an API token):

```bash
pip install kaggle
# put your kaggle.json at ~/.kaggle/kaggle.json (chmod 600)
mkdir -p data/full
cd data/full
kaggle competitions download -c jigsaw-unintended-bias-in-toxicity-classification
unzip jigsaw-unintended-bias-in-toxicity-classification.zip
```

After unzipping you should have, under `data/full/`:

| File | Rows | Notes |
|---|---|---|
| `train.csv` | ~1.80 M | training data with `target` (continuous toxicity), `comment_text`, 24 identity columns, sub-toxicity labels, metadata |
| `test.csv` | ~97 k | test inputs only (`id, comment_text`) — no labels |
| `test_public_expanded.csv` | ~97 k | public test set with `toxicity` + identity labels |
| `test_private_expanded.csv` | ~97 k | private test set with `toxicity` + identity labels |
| `sample_submission.csv` | ~97 k | competition submission template |
| `all_data.csv`, `*_individual_annotations.csv` | — | auxiliary, not used by the pipeline |

Our pipeline:
- trains/validates from `train.csv` only (with a stratified 95/5 split)
- evaluates on the **union of `test_public_expanded.csv` + `test_private_expanded.csv`** — these are the only labelled test files

## Re-generating the tiny subset

```bash
python scripts/make_tiny_subset.py --src data/full --out data/tiny
```

Deterministic (seeded). Run this if you want a different size or seed.
